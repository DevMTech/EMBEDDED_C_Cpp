Git initialization
==================
How can you start a project using Git?
'git init' command on an empty directory
'git init' command in a directory containing project files already
cloning an existing project from somewhere else (Github)

$ git init [project-name]	// project-name parameter is optional. 
			// If not supplied, Git will initialize the current directory.

STARTING WITH FRESH / NEW GIT REPO
$ pwd

$ cd git-projects/			// C:\Users\erdma\git-projects

$ git init git-demo // STARTING WITH FRESH / NEW GIT REPO / PROJECT

$ ls -a


WORKING WITH EXISTING GIT REPO
$ pwd

$ cd git-projects/

$ cd git-demo/

$ ls

$ git init			// WORKING WITH EXISTING GIT REPO / PROJECT 


$ notepad README.md // make changes to the README.md, save & exit

$ git status		// Creating/Editing File/s in Working Directory
				(use "git add <file>..." to update what will be committed)
				(use "git restore <file>..." to discard changes in working directory)

$ git add README.md // Adding new changes in File/s to Git Staging Area (Git Index)
				(use "git restore --staged <file>..." to unstage)

$ git status   		// File is in Git Staging Area, ready to be commited  

$ git commit -m "Initial Commit" // Commiting all changes in Files currently in Git's Staging Area

$ git status		// Working Directory has no uncommited Files

$ git commit -am "YOU CAN BOTH ADD TO STAGING AREA & COMMIT USING THIS SHORTCUT COMMAND"

// You cannot use the git commit -a shortcut on new files ??

$ clear

$ git add .

$ git commit -m "ALL MODIFIED FILES COMMITED"

$ git log // ENTER TO SEE MORE | 'q' or 'z' TO EXIT

$ git log --color --decorate --graph --oneline


 correct syntax for adding newly modified files to Git's staging area
 git add file-name
 git add .
 git commit -a
 git add -A
 

STEPS TO MAKE CHANGES TO A FILES AND ROLL BACK FROM STAGING AREA
TO PREVIOUS VERSION BEFORE THE CHANGES WERE MADE

$ notepad README.md // MAKE CHANGES TO README.md 

$ git add README.md  // ADDED TO STAGING AREA

$ git status

$ git reset HEAD README.md // UNSTAGE FILES
				Unstaged changes after reset:
				M       README.md

$ git status

$ notepad README.md 

$ git checkout -- README.md  // ROLL BACK TO PREVIOUS VERSION OF FILE

$ git status

$ notepad README.md 


$ git log -2

$ git stash // to reject all changes





DELETE A FILE THA THAS BEEN COMMITED BY MISTAKE
$ notepad debug.log // EDIT THE FILE

$ git status
				On branch master
				Untracked files:
				  (use "git add <file>..." to include in what will be committed)
						debug.log

$ git add debug.log

$ git status
				On branch master
				Changes to be committed:
				  (use "git restore --staged <file>..." to unstage)
						new file:   debug.log
		
$ git commit -m "debug.log file commited by mistake"
				[master 914dd1b] debug.log file commited by mistake
				 1 file changed, 1 insertion(+)
				 create mode 100644 debug.log
 
$ git rm debug.log

$ git status
				On branch master
				Changes to be committed:
				  (use "git restore --staged <file>..." to unstage)
						deleted:    debug.log
		
$ git commit -m "removing debug.log"
				[master fd80f1f] removing debug.log
				 1 file changed, 1 deletion(-)
				 delete mode 100644 debug.log

ADDING FILES TO .gitignore FILE, SO THAT THEY ARE IGNORED WHEN COMMITING 
The .gitignore file in your repository must be version controlled like any other file in Git

$ notepad app.log

$ git status
				On branch master
				Untracked files:
				  (use "git add <file>..." to include in what will be committed)
						app.log
				nothing added to commit but untracked files present (use "git add" to track)

$ notepad .gitignore // *.log 

$ git status 
				On branch master
				Untracked files:
				  (use "git add <file>..." to include in what will be committed)
						.gitignore
				nothing added to commit but untracked files present (use "git add" to track)

$ git add .gitignore

$ git status
				On branch master
				Changes to be committed:
				  (use "git restore --staged <file>..." to unstage)
						new file:   .gitignore
		
$ git commit -m "Adding .gitignore file"
				[master b1f27b9] Adding .gitignore file
				 1 file changed, 1 insertion(+)
				 create mode 100644 .gitignore
